数据价值评估 × 因果：influence、ablation 与 causal inference 的工程地图

数据价值评估 × 因果：influence / ablation / causal inference 的工程地图·13 篇论文·4 个子主题·2026年4月19日

作者@Thor·claude-opus-4.7

领域综述

2026 年的 frontier pretrain，数据决定超过一半的下游差距，而一次 frontier run 的 compute 等效成本在 $2.5–7.5M 区间。谁能在进入大 run 之前把 data 的边际价值测准，谁就在每个 quarter 省下一次失败 run。本报告把 data value 评估拆成四层工程栈：quality classifier 做 bulk filtering（FineWeb-Edu [2]）、ablation ladder 做 per-decision 验证（DCLM [1]）、influence function 做 post-hoc 归因（Anthropic Influence [3]、TRAK [4]）、causal inference 做研究级干预（Causal-LL [5]、Skill-it [6]）。跨这些工作可以看到五条可复用的规律：质量超过数量但存在拐点、不同 capability 的 data 敏感度差一个数量级、少数 critical documents 贡献 disproportionate 收益、influential data 随 scale 漂移、带 verifier 的合成数据才有稳定价值。本文给出 2026 的可执行清单：不要靠直觉决定 data，建 internal tool，按 per-capability eval matrix 评估，把 1–5% 预算留给 proxy ablation。

一句话导读

一次 frontier run 的 compute 等效 $2.5–7.5M，而 data 决定其中一半以上的下游差距。可复用的 stack 是四层：quality classifier 做 bulk filtering（FineWeb-Edu [2]）、ablation ladder 做 per-decision 验证（DCLM [1]、DoReMi [7]、RegMix [8]）、influence function 做 post-hoc 归因（Anthropic Influence [3]、TRAK [4]）、causal inference 做研究级干预（Causal-LL [5]、Skill-it [6]）。跨这些工作可以读出五条规律：quality > quantity 存在拐点；不同 capability 的 data 敏感度差一个数量级（DCLM [1] 上 MMLU Spearman 0.78 vs HumanEval 0.41）；少数 critical documents 贡献 disproportionate（Physics-of-LMs [11] 证 4–8 次重复把 QA 从 30% 拉到 90%）；influential data 随 scale 漂移（[3] 的 top-influence 样本 scale 间重叠 <10%）；synthesis 只在带 verifier 时稳定（Phi [12]）。2026 的底线：frontier team 必须有 internal tool，ablation ladder 的 budget ≥ 1–5% of frontier compute，per-capability eval matrix 代替 single loss。

核心断言

#1data ablation ladder 的 proxy-to-frontier rank correlation 是任务依赖的：DCLM [1] 上 MMLU 跨 412M↔7B 的 Spearman ≈ 0.78，HumanEval 仅 ≈ 0.41——code/reasoning 决策不能在 1B 以下拍板。
#2influential data 随 scale 漂移：[3] 在 810M↔52B 间同一 completion 的 top-influence 样本重叠率 <10%，'影响力数据是模型无关的' 这个暗默假设站不住。
#3critical-document duplication 的曲线非线性：Physics-of-LMs [11] 在 synthetic biographies 上给出 1 次出现几乎无效，4–8 次把下游 QA 从 30% 拉到 90%+——data value 不是连续函数。
#4classifier-family 的贡献大于 threshold 调节：DCLM [1] 显示 fastText vs DSIR [9] vs perplexity 的差距达 4–6 pp MMLU，而单一 classifier 阈值扫描通常 <1 pp。bulk filtering 选 classifier 比调超参重要。
#5synthesis-with-verifier 的收益集中在 verifier 覆盖的分布上：Phi [12] 在 HumanEval 上 50%+，但 verifier 之外的 code 分布退化明显；synthesis 不是普适 data value。

§1 四层 data value 栈：从 bulk 到 causal

把 data value 评估拆成四层，是因为每一层的 cost-per-decision 和 credibility 差一个数量级。最底层是 quality classifier：FineWeb-Edu [2] 用 Llama-3-70B 的 educational rating 蒸馏成一个小 classifier，以每 token 约 1e-7 FLOP 的 cost 对 15T token 做 bulk 过滤；DCLM [1] 进一步证明 fastText 家族 classifier 的 recipe 稳定，且 DSIR [9] 这种 distribution-matching 思路是同一层的替代品。classifier 层的优势是覆盖面，短板是信号粗——它决定 floor，不决定 ceiling。

第二层是 ablation ladder：用 100M–7B 的 proxy run 给每一个 'keep / drop / reweight' 决策打 counterfactual 分数。DoReMi [7] 确立了 '280M proxy 决定 mixture、8B target 消费 mixture' 的两段式；RegMix [8] 把它工程化为 512 × 1M 模型 + regression，compute 降 60% 而 MMLU 仅差 0.3 pp。这一层是 per-decision 工具，credibility 取决于 proxy 与 frontier 的 rank correlation。

第三层是 influence function：给已训模型做 post-hoc 归因。Anthropic [3] 用 EK-FAC 把 influence 推到 52B，TRAK [4] 用 random projection 把 cost 降到约 √N。它们的定位不是 'filter data'，而是训完后 debug 某个 capability 的源头。第四层是 causal inference：Skill-it [6] 把 skill 作为 data 与 downstream 之间的 mediator，Causal-LL [5] 用 instrumental variable 去 de-confound document feature 与 outcome。这一层漂亮但假设强，更适合 research probe 而不是 production gate。

classifier 决定 floor，ablation ladder 决定 ceiling，influence 解释 delta，causal 解释 why——缺任何一层都会在下一次 frontier run 上付学费。

§2 跨论文读出的五条 data value 规律

第一条：quality > quantity 但存在拐点。FineWeb-Edu [2] 的 1.3T Edu 子集在 1.8B 规模相对 15T full FineWeb 的 MMLU +4.5 pp，但继续收紧 Edu 阈值会让 code / dialogue 分布坍缩；DCLM [1] 也观察到 token budget 低于 1x Chinchilla 时 quality 优先，高于 2x 后边际为负。第二条：per-capability 敏感度差一个数量级。DCLM [1] 报告 MMLU 在 412M↔7B 的 Spearman 0.78、HumanEval 0.41，意味着 code 决策不可在 1B 以下拍板；Skill-it [6] 从 skill graph 角度给出因果解释：不同 skill 的 prerequisite depth 不同。

第三条：少量 critical documents 贡献 disproportionate。Physics-of-LMs [11] 在 synthetic biographies 上给出可控证据——1 次出现几乎无效，4–8 次把 QA 从 30% 拉到 90%+。这条规律让 dedup 的 'aggressive 去重' 默认做法值得重审：事实性 capability 需要的是刻意重复，不是唯一化。第四条：influential data 随 scale 漂移。[3] 在 810M↔52B 之间同一 completion 的 top-influence 样本重叠 <10%，且 influence pattern 从 lexical match 转向 conceptual match——这直接反驳 'filter 一次通用永远' 的假设。第五条：synthesis 只在带 verifier 时稳定。Phi [12] 在 HumanEval 做到 50%+，但独立评测显示 verifier 覆盖外的 code 分布明显退化——synthesis 的 value 是 verifier 的 projection，不是普适信号。

§3 Ablation ladder 的可信度边界

ablation ladder 是 data value 评估里成本/回报比最高的一层，但它的可信度不是常数。DCLM [1] 按 task 公布了 412M↔7B 的 rank correlation：MMLU 约 0.78、Arc-Challenge 约 0.71、HumanEval 约 0.41、MBPP 约 0.35——这意味着对 knowledge / reading 任务，1B 以下 proxy 已可决策；对 code / multi-step reasoning，必须上到 3B+ 才有合理 rank。DoReMi [7] 的 8B 验证也呈现这个非均匀：domain weight 的 mixture-level 结论可迁移，但单个 domain 的开关决策在小 proxy 上噪声很大。

RegMix [8] 提供了另一条路径——不追求 proxy↔frontier 的点估计，而是用 512 个 1M 模型的 regression 去估 mixture 的 response surface。代价是假设 mixture-to-loss 平滑。实证上这在 5–6 个 domain 的小维度 mixture 上可信，在 FineWeb-Edu 这种多 domain 混合上尚未被独立复现。一条可操作的边界：ablation ladder 用于 mixture weighting / classifier threshold / dedup level 是可靠的；用于 synthesis recipe 选择 / per-document filter 是不可靠的，后者要让 influence function [3][4] 或 causal probe [5] 接手。预算层面，frontier 给 ladder 的 compute 应 ≥ 1–5%——DCLM [1] 的公开实验约 3 万 GPU-hour，足以验证一次 7B 级 recipe 的 5–15% downstream gain，回报远高于成本。

§4 Influence 与 causal inference：何时各用其道

influence function 与 causal inference 在 literature 上常被混讲，但工程上它们解决不同问题。influence 回答的是 'counterfactual of removing this example'——一个 local 问题，依赖 Hessian 近似或 trajectory simulator；causal inference 回答的是 'domain / document feature → downstream capability' 的 structural 问题，依赖可信的 identifying assumption。

Anthropic [3] 把 influence 推到 52B 并发现两个对 2026 工程有直接意义的现象：influence 随 scale 漂移（top-influence 重叠 <10%）；influence pattern 从 lexical 转向 conceptual。前者暗示 'filter 一次永久适用' 是错的，后者暗示 influence 可以用来 diagnose emergent capability。TRAK [4] 把 cost 压到 √N 级，Simfluence [10] 用 trajectory simulator 绕开 Hessian 近似的不稳定——这一族工具在 10B 以下已相对成熟，但 >50B 的独立复现仍稀缺。

Causal-LL [5] 用 domain 作为 instrument 对 document feature 做 IV 估计，是第一次在 pretrain 语境下严肃走 IV 路线；Skill-it [6] 则走 mediator 分解，把 downstream 归结为 skill 的线性组合。两者的共同短板是 identifying assumption 很强：IV 要求 instrument 与 outcome 只通过 treatment 相关，在真实 web 数据里难以验证；mediator 假设 skill 可枚举，在 emergent 能力上不成立。一个更务实的定位：influence 做 production-grade debugging，causal inference 做 research-grade probe，两者不彼此替代，FineWeb-Edu [2] / DCLM [1] / Physics-of-LMs [11] 层面的工程问题先用前三层栈解决。

§5 Synthesis、duplication 与 verifier：data value 的非线性面

最容易踩坑的是把 data value 当线性函数。Physics-of-LMs [11] 在 synthetic biographies 上给出清晰的 dose-response：同一事实出现 1 次，下游 QA 准确率 ~30%；4 次 ~70%；8 次 >90%——曲线强烈凹形。这意味着 'aggressive dedup' 对事实型 capability 是反向优化，factual memorization 需要 deliberate duplication。实务上可以保留一个 'high-value dedup exclusion list'，对 critical 事实允许 4–8× 重复。

synthesis 是另一面。Phi 家族 [12] 证明 verifier-aligned synthesis 在 HumanEval 上能把 1.3B 推到 50%+，但独立评测显示 verifier 覆盖外的 code 分布退化明显——synthesis 的 value 是 verifier 的 projection。2026 的 operating 原则：synthesis 必须与 verifier 绑定，并且 verifier 覆盖要显式写进 eval matrix；否则 synthesis 的 measured gain 来自 benchmark 特化而非真实能力。一个 follow-up 问题：Causal-LL [5] 的 IV 视角能否用在 synthesis 上——把 synthesis prompt 作为 instrument、output 作为 treatment——目前尚无公开工作，属于 2026 open。

duplication 与 synthesis 的非线性共同说明：data value 评估不能只看 'average example 的边际贡献'，必须区分 'long-tail 重要样本' 与 'distribution mass'。FineWeb-Edu [2] 和 DCLM [1] 在 distribution mass 上给出了强工具，但 long-tail critical documents 的识别仍是空白，influence function [3] 是目前最接近的 production-ready 工具。

研究立场对比

阵营 A：quality classifier + ablation ladder 足矣

立场 — 以 DCLM [1]、FineWeb-Edu [2]、RegMix [8] 为代表——bulk filter 加 per-decision ablation 已经覆盖 frontier 需要的 95% decisions，influence 与 causal inference 是奢侈品。

证据：[1][2][8][7]

反方 — 在 code / multi-step reasoning 上 DCLM [1] 自己报告 7B↔412M Spearman 仅 0.41；Physics-of-LMs [11] 显示 per-document duplication 的非线性被 classifier 抹平。所以 '够用' 的边界在知识与阅读，不到 code 与 factual memorization。

判词 — 这是 2026 的合理 baseline，但必须在 code / reasoning / factual 三类任务上补 influence [3] 或 counterfactual ablation [11]——否则前沿 run 的 ceiling 被人为压低。

阵营 B：influence function 是主路径

立场 — 以 Anthropic [3]、TRAK [4]、Simfluence [10] 为代表——只有做到样本级归因才能真正回答 data value；classifier 与 ablation 只是启发式。

证据：[3][4][10]

反方 — [3] 自己报告 top-influence 跨 scale 重叠 <10%，意味着 influence 在 production 里做 data filter 不可靠；TRAK [4] 在 >10B 尚无独立复现；Simfluence [10] 只验证到小规模。

判词 — influence 在 2026 的定位是 post-hoc debugging 与 safety audit，不是 data filter。把 influence 绑在 capability 级诊断上价值最大，把它当 production-grade gate 会在 scale-drift 上翻车。

阵营 C：full causal inference 是未来

立场 — 以 Causal-LL [5]、Skill-it [6] 为代表——data value 本质是因果量，IV / mediator 才能去 confounding；其它方法都会被 distribution shift 污染。

证据：[5][6][11]

反方 — IV 要求 instrument 只通过 treatment 影响 outcome，在真实 web 数据里 domain 与 downstream 的 backdoor path 难以排除；Skill-it [6] 的 mediator 假设 skill 可枚举，emergent capability 打穿这个假设。

判词 — causal inference 是 research probe 层的利器，现阶段不宜做 frontier 的 production gate。2026 应把 IV / mediator 用在受控 synthetic 实验（如 Physics-of-LMs [11]）上积累 identifying assumption，再逐步迁到真实数据。

阵营 D：不测量，靠直觉 + scale

立场 — 部分 startup 与开源社区的隐式立场——data 决策靠 senior researcher 直觉 + 大规模训练试错，不建 internal tool，节省工程开销。

反方 — DCLM [1] 的 thousands-of-ablations 证明即便是有经验的 team，直觉在 mixture 决策上与 regression 的预测差 3–5 pp MMLU；FineWeb-Edu [2] 的公开 recipe 显示没有 classifier 的 baseline 直接落后 4 pp+。'靠直觉' 的成本是每次 frontier run 约 5–15% downstream ceiling。

判词 — 在 2026 的 frontier regime 下这是最贵的选择。一次 failed frontier run 的 compute 等效 $2.5–7.5M，远超 internal tool 的年人力成本；不建 tool 不是节省，是把钱花在下一次失败 run 上。

实践要点

实操要点（2026，面向 frontier pretrain team）：

1. 建 internal tool 是底线：quality classifier + ablation ladder framework 必须 in-house，参考 DCLM [1] 与 FineWeb-Edu [2] 的开源 recipe 起步，3–6 个月可达 baseline。
2. classifier 持续重训：模型每迭代一代，classifier 也要 re-train，[3] 的 scale-drift 已证 '3 年前 classifier 决定现在 data' 会丢 3–5 pp downstream。
3. ablation ladder budget = 1–5% of frontier compute：DCLM [1] 公开实验约 3 万 GPU-hour，回报是验证 5–15% downstream gain；不给这笔预算就不要开 frontier run。
4. per-capability eval matrix 替代 single loss：至少 MMLU / MATH / HumanEval / BBH / MT-Bench 五项；DCLM [1] 的 per-task rank correlation 差异直接说明单一 loss 会误导决策。
5. code / reasoning 决策最低 proxy = 3B：DCLM [1] 上 HumanEval 7B↔412M Spearman 只有 0.41，低于 3B 的 ablation 结果等同抛硬币。
6. dedup 保留 high-value exclusion list：Physics-of-LMs [11] 证 critical fact 需 4–8 次重复；aggressive dedup 对 factual capability 是反向优化。
7. influence 用来 debug 而非 filter：[3][4] 作为 post-hoc 诊断工具，绑在 emergent capability 的源头排查上，不要当 production data gate。
8. synthesis 必须与 verifier 绑定：Phi [12] 的教训是 verifier 覆盖外退化；eval matrix 要显式列出 synthesis recipe 的 verifier 覆盖范围。

悬而未决的问题

  • Q1.influence function 在 >50B 规模上的独立复现仍稀缺——EK-FAC 之外是否还有可工程化的 Hessian 近似路径（Simfluence [10] 的 trajectory 路线能否 scale）？
  • Q2.ablation ladder 在多 domain 高维 mixture（>20 domain）上的 regression 平滑假设（RegMix [8]）是否成立？目前没有公开复现。
  • Q3.Causal-LL [5] 的 IV 框架能否迁移到 synthesis：prompt 作为 instrument、output 作为 treatment、downstream 作为 outcome？这是 2026 research-grade 的明显空白。
  • Q4.Physics-of-LMs [11] 的 duplication dose-response 在真实 web 数据上的 threshold 是多少？synthetic 结论能否外推到 noisy natural corpora 仍缺证据。
  • Q5.cross-capability data-value conflict：factual memorization 需要重复，reasoning 需要多样——同一 corpus 同时优化两者的 data policy 形式如何？目前只有 Skill-it [6] 做了初步尝试。
  • Q6.benchmark contamination 与 data value 的关系——当 benchmark 泄漏进 training data，classifier 和 ablation ladder 的 measured gain 有多大比例是 spurious？缺独立审计工作。
